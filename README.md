## http status code

1xx: Informational (ข้อมูลเบื้องต้น)เป็นกลุ่มแจ้งสถานะชั่วคราวว่าเซิร์ฟเวอร์ได้รับคำขอแล้ว และกำลังรอการดำเนินการขั้นต่อไป

100 Continue: เซิร์ฟเวอร์ได้รับคำขอส่วนแรกแล้ว ให้ผู้ใช้ส่งข้อมูลที่เหลือต่อได้เลย

101 Switching Protocols: เซิร์ฟเวอร์กำลังเปลี่ยนผ่านโปรโตคอลตามที่ผู้ใช้ร้องขอ (เช่น เปลี่ยนจาก HTTP ไปเป็น WebSocket)

2xx: Success (การทำงานสำเร็จ)เป็นกลุ่มที่แสดงว่าเซิร์ฟเวอร์ได้รับ รับทราบ และประมวลผลคำขอเรียบร้อยแล้ว

200 OK: คำร้องขอสำเร็จลุล่วงด้วยดี เป็นรหัสที่เกิดขึ้นบ่อยที่สุดเวลาเราเปิดหน้าเว็บปกติ201 Created: คำร้องขอสำเร็จ และเซิร์ฟเวอร์ได้สร้างข้อมูลใหม่ขึ้นมาแล้ว (มักเกิดหลังจากการส่งข้อมูลแบบ POST)

202 Accepted: เซิร์ฟเวอร์ยอมรับคำขอแล้ว แต่ยังประมวลผลไม่เสร็จ (มักใช้กับงานเบื้องหลังที่ใช้เวลานาน)

204 No Content: ทำงานสำเร็จเรียบร้อยดี แต่ไม่มีเนื้อหาข้อมูลส่งกลับมาให้ (เช่น กดบันทึกข้อมูลสำเร็จ แต่ไม่มีหน้าเพจใหม่ให้แสดง)

3xx: Redirection (การเปลี่ยนเส้นทาง)กลุ่มนี้บอกให้เบราว์เซอร์หรือแอปพลิเคชันต้องขยับไปทำขั้นตอนอื่นต่อ หรือเปลี่ยนไปยัง URL อื่นเพื่อให้งานเสร็จสิ้น

301 Moved Permanently: หน้าเว็บนี้ถูกย้ายไปยัง URL ใหม่ถาวรแล้ว (ดีต่อการทำ SEO เพื่อส่งคะแนนไปยังลิงก์ใหม่)

302 Found: หน้าเว็บถูกย้ายไปที่อื่นชั่วคราว (ปัจจุบันมักแนะนำให้ใช้ 307 แทน)

304 Not Modified: ข้อมูลหน้าเว็บไม่ได้อัปเดตใหม่ ให้เบราว์เซอร์ดึงข้อมูลเก่าจากแคช (Cache) มาใช้งานได้เลยเพื่อความรวดเร็ว

4xx: Client Error (ข้อผิดพลาดจากฝั่งผู้ใช้)กลุ่มที่พบบ่อยเมื่อทางฝั่งเราส่งคำขอไม่ถูกต้อง ไม่มีสิทธิ์เข้าถึง หรือพิมพ์ URL ผิด

400 Bad Request: คำขอไม่ถูกต้องหรือ syntax ผิดพลาด ทำให้เซิร์ฟเวอร์อ่านไม่เข้าใจ401 Unauthorized: ไม่พบตัวตน จำเป็นต้องเข้าสู่ระบบ (Login) หรือส่ง Token ยืนยันตัวตนก่อน403 Forbidden: เซิร์ฟเวอร์รู้จักคุณ แต่คุณไม่มีสิทธิ์เข้าถึงไฟล์หรือหน้านี้

404 Not Found: ยอดฮิตที่สุด หมายถึงหาหน้าเว็บหรือไฟล์นี้ไม่เจอ (URL ผิด หรือข้อมูลถูกลบไปแล้ว)

405 Method Not Allowed: ใช้คำสั่งผิดประเภท เช่น หน้าเว็บนี้อนุญาตให้ใช้ GET เพื่ออ่านได้อย่างเดียว แต่คุณดันส่ง POST เพื่อเขียนข้อมูลเข้ามา

429 Too Many Requests: คุณส่งคำร้องขอถี่หรือมากเกินไปในเวลาอันสั้น (เซิร์ฟเวอร์บล็อกเพื่อป้องกันการโดนยิงเว็บ)

5xx: Server Error (ข้อผิดพลาดจากฝั่งเซิร์ฟเวอร์)ฝั่งผู้ใช้ส่งคำขอมาถูกต้องทุกอย่างแล้ว แต่ระบบหลังบ้านหรือเซิร์ฟเวอร์พังจนไม่สามารถตอบสนองได้

500 Internal Server Error: ข้อผิดพลาดทั่วๆ ไปในฝั่งเซิร์ฟเวอร์ เช่น โค้ดหลังบ้านบั๊ก หรือฐานข้อมูลล่มโดยบอกสาเหตุเจาะจงไม่ได้

502 Bad Gateway: เซิร์ฟเวอร์ที่เป็นด่านหน้า (Proxy/Gateway) ได้รับการตอบกลับที่ผิดพลาดมาจากเซิร์ฟเวอร์หลักด้านหลัง

503 Service Unavailable: เซิร์ฟเวอร์ไม่พร้อมใช้งานชั่วคราว (ส่วนใหญ่เกิดจากคนเข้าเว็บเยอะเกินจนเซิร์ฟเวอร์รับไม่ไหว หรือระบบกำลังปิดปรับปรุง)

504 Gateway Timeout: เซิร์ฟเวอร์ที่เป็นด่านหน้า รอการตอบกลับจากเซิร์ฟเวอร์หลักนานเกินไปจนหมดเวลา
